HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user.

CSS

  • (1) inline CSS (2) internal CSS (3) external CSS style sheet
  • Separation of Concerns: separating code into different files based on use
  • Do Not Repeat Yourself (DRY): a coding principle that encourages developers to reuse or share pieces of code in order to reduce the number of lines of code that need to be written or maintained.
  • CSS Rule: Selector { property: value; color: blue}
  • declaration: the {property: value} pair
  • margin: space OUTSIDE the element
  • boarder: the outermost part of an element
  • padding: space around the content INSIDE the element
    • content: the innermost part of an element
  • *: a wild card selector: the rule apple to ALL elements

Git

  • git clone "copy here the address from GitHub": clone the Github project to local git environment
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: add all files to the staging area of Git
  • git commit -m "description of the changes": -m associate a message with the commit
  • git pull origin main: pull the main branch from GitHub into my feature branch before pushing up any code
  • git push origin feature/add-html: push up all the changes

JavaScript

  • All primitive data type are immutable
  • Only Object data type is mutable
